### Next.js

ここ一週間でNext.jsに関する最大の話題は、深刻なセキュリティ脆弱性（CVE-2025-55182およびCVE-2025-66478）の発見と修正です。これにより、リモートコード実行（RCE）のリスクが生じ、多くの開発者がアップデートを急いでいます。例えば、[@duborges](https://x.com/duborges/status/1997293892090183772)は自身のサーバーがハッキングされた体験を共有し、Dockerコンテナのroot権限が原因だったと警告しています。 また、[@graphtheory](https://x.com/graphtheory/status/1997521557996847283)はこの脆弱性を「Next.jsの贈り物」と皮肉り、静的サイトへの移行を推奨。

一方で、Next.jsからの移行トレンドも目立ちます。[@theo](https://x.com/theo/status/1997406196660400228)はT3 ChatをNext.jsからTanStack Startへ移行した理由を説明し、デバッグのしやすさを強調。 [@lyf1point0](https://x.com/lyf1point0/status/1997538724910153744)も同様に、TanStackへの移行を議論しています。

Next.js 16の新機能も注目され、[@DThompsonDev](https://x.com/DThompsonDev/status/1997527156126347588)はNext.js 15/16を使った新サイトのスタックを共有。 ウェブ記事では、[@rtsekov](https://medium.com/@rtsekov/critical-alert-your-next-js-project-could-be-owned-by-a-single-http-request-2adedcb6929f)がHTTPリクエストによる乗っ取りの危険性を指摘しています。

### Tailwind CSS

Tailwind CSS v4の新機能が活発に議論されており、[@saswatapal](https://dev.to/saswatapal/tailwind-css-v4-why-i-chose-css-first-config-over-styled-components-270f)はv4のCSS優先設定がStyled Componentsより10倍速いビルドを実現するとベンチマークを紹介。 Xでは、[@KharayKrayKray](https://x.com/KharayKrayKray/status/1997357948335473114)がTailwindを諦めて素のCSSに戻りたくなるジレンマを投稿し、多くの共感を呼んでいます。

スタックとしての使用例も多く、[@DThompsonDev](https://x.com/DThompsonDev/status/1997527156126347588)はNext.jsとTailwind v4を組み合わせた新サイトをプレビュー。 [@rsStats_](https://x.com/rsStats_/status/1997366680368878005)はTailwind + shadcnを使ったAIチャットテンプレートをオープンソース公開。 記事では、[@elvissautet](https://dev.to/elvissautet/tailwind-css-won-the-war-but-were-the-losers-4877)がTailwindの普及が開発者の負けだと批判的に分析しています。

### shadcn/ui

shadcn/uiの新コンポーネント追加が話題で、[@mdnlabs](https://x.com/mdnlabs/status/1997511853606679003)はNext.js + shadcnを使ったSaaSスタックを共有。 [@gravyskyy](https://x.com/gravyskyy/status/1997487954059788680)はshadcnを「UI/UXのBanksy」と称賛。

テンプレートや代替品も注目され、[@saltcod](https://x.com/saltcod/status/1997447856769360100)はNext.js 16との統合を議論。 [@ttyyor](https://x.com/ttyyor/status/1997408684771881120)はshadcnを使ったURL短縮ツールのプロトタイプを投稿。 記事では、[@javascript-in-plain-english](https://medium.com/javascript-in-plain-english/shadcn-just-dropped-7-new-components-and-theyre-absolutely-insane-562d2aeb36a9)が7つの新コンポーネントを絶賛。

### React Aria Components

アクセシビリティの強化が主な話題で、[@hagipen1](https://x.com/hagipen1/status/1997156831152820348)はReact Aria Componentsを使ったWeb a11y対応の記事を共有。 [@suk](https://x.com/suk/status/1997066073196450132)も同様の記事を投稿。

ライブラリとして、KanpekiのようなReact Ariaベースのアクセシブルコンポーネントが登場。 記事では、[@jqueryscript](https://dev.to/jqueryscript/kanpeki-accessible-react-components-built-on-react-aria-464c)がKanpekiを紹介。 全体として、2025年のReactトレンドとしてアクセシビリティ重視が強調されています。

Prisma v7の課題とスタック統合が議論され、[@techtalkjp](https://x.com/techtalkjp/status/1997530319621427306)はbetter-authとの組み合わせで管理UIを再構築。 [@DThompsonDev](https://x.com/DThompsonDev/status/1997527156126347588)はNext.js + Prismaのスタックを共有。

### Biome

Biome v2のリリースが注目で、[@techtalkjp](https://x.com/techtalkjp/status/1997530319621427306)はv2へのアップデートを報告し、設定ファイルの簡単移行を称賛。 [@saswatapal](https://dev.to/saswatapal/why-i-chose-biome-over-eslintprettier-20x-faster-linting-one-tool-to-rule-them-all-10kf)はBiomeがESLint+Prettierより20倍速いとベンチマーク。

記事では、[@thatanjan](https://dev.to/thatanjan/i-replaced-eslint-prettier-with-this-fkn)がBiomeをESLint/Prettierの代替として推奨。 2025年のトレンドとして、BiomeのHTMLサポートなどが期待されています。
