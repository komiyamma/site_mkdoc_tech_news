2026年4月19日〜26日の**Firebase**、**GCP（Google Cloud）**、**Vercel（v0含む）**、**Cloudflare**、**Supabase**、**Azure**の新情報を、X（Twitter）の全ユーザー投稿およびウェブ検索から収集・整理しました。古い情報は除外し、公式発表や信頼できる更新を中心にまとめています。同一アカウントからの引用は3件以内に抑え、分散させています。

### Firebase / GCP（Google Cloud）
Cloud Next '26（4月22日開催）でFirebaseとGCPの大規模アップデートが複数発表されました。  
- Firebase Data Connectが「**Firebase SQL Connect**」に名称変更。Cloud SQL（PostgreSQL）と連携し、リアルタイム同期・オフラインキャッシュ・ネイティブSQLをサポート。無償トライアルで利用可能になりました。  
- **Firestore Enterprise**でpipeline queriesが一般提供（GA）。全文検索、地理空間クエリ、subqueriesによるJOIN、DML対応など新クエリエンジンが強化されました。  
- FirebaseとApplication Design Centerの統合により、モバイル/ウェブアプリのリソース（Firestore、Auth、AI Logicなど）をGoogle Cloudインフラと一元管理可能に。  
- Firestore向けAI-powered MQLクエリ生成（Preview）や、AI Logicのmulti-turnサポート追加、JavaScript SDK v12.12.1リリースなども同時発表。  

Firebase公式エンジニアの**[@sgnagnarella](https://x.com/sgnagnarella/status/2048060295537373214)**がNextでの発表リキャップ動画を共有。  
また、Google SecOpsに**Agentic Defence Security**の新目的別エージェント（Threat Hunting、Detection Engineering、Third-Party Context）が追加され、WizのGoogle Cloud完全統合も発表されました。**[@meshalabhjain](https://x.com/meshalabhjain/status/2048253631720517801)**が詳細を投稿。

### Vercel（v0含む）
**4月19日**に重大なセキュリティインシデントを公式公表。第三者AIツール（Context.ai）のOAuth侵害により内部システムに不正アクセスが発生し、非sensitive環境変数が影響を受けました。顧客に対し即時ローテーションを強く推奨し、CEOが攻撃経路を詳細説明。  
これを機に、環境変数管理の強化（デフォルト設定強化、チーム全体概要表示、activity logの深リンク・高密度化）などの製品改善を展開中です。

その他製品アップデート：  
- **Vercel Workflows**が一般提供（GA）。長時間実行の耐久性が高いエージェント/バックエンドを簡単に構築可能に。  
- Elastic build machinesがGA（新Proチームのデフォルト適用、Turbo build価格16%引き下げ）。  
- AI Gatewayに**GPT-5.5**、**DeepSeek V4**、**GPT Image 2**などの新モデルを追加。  

v0特有の新機能発表は今週確認されませんでした（ドキュメント更新などの言及はありましたが新機能ではありません）。
